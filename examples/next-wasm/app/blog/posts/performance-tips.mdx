# Performance Tips

## Use Server Components

<Callout type="warning">
Server Components eliminate client-side JavaScript for MDX rendering.
</Callout>

```typescript
import { MDXServerComponent } from 'bunny-next/rsc';

export default async function Page() {
  return <MDXServerComponent source={content} />;
}
```

## Enable Caching

<Counter initialCount={50} />

For repeated content, use `serializeWithCache()`:

```typescript
import { serializeWithCache } from 'bunny-next';
const mdxSource = await serializeWithCache(content);
```

## Performance Comparison

| Approach | Client JS | Best For |
|----------|-----------|----------|
| Local MDX | Minimal | Static content |
| MDXRemote | Medium | Dynamic content |
| MDXServerComponent | **Zero** | SEO & Performance |
